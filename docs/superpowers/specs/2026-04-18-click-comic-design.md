# Click Comic — Design Spec

_2026-04-18_

## Overview

A click comic is a sequential panel viewer: images advance one at a time on user input, loop or stop when finished. The system has two outputs from one Vite project:

1. **A static website** — gallery of all comics, each viewable in the browser
2. **An ESM module** — `click-comic.es.js` that any game can import and use as a cutscene

---

## Architecture

### Project Structure

```text
click-comic-experiment/
  src/
    lib/
      ClickComic.js       ← core viewer class (also the module export)
      Gallery.js          ← gallery page logic (reads comic manifests, renders grid)
    main.js               ← gallery site entry point
    viewer.js             ← viewer page entry point
  public/
    comics/
      shadows-of-lilim/
        comic.json
        01.png … 45.png
      <next-comic>/
        comic.json
        01.png …
  index.html              ← gallery page template
  viewer.html             ← viewer page template
  vite.config.js
  package.json
  dist/                   ← built output (deploy this folder)
    index.html
    viewer.html
    click-comic.es.js     ← importable ESM module for games
    comics/               ← copied as-is from public/
```

### Build Outputs

Vite is configured in library mode with a secondary MPA (multi-page app) build:

- **Static site**: `index.html` (gallery) + `viewer.html` (viewer) — deploy `dist/` to any static host
- **ESM module**: `click-comic.es.js` — copy into any game project and import directly

### Local Development

```bash
npm run dev    # Vite dev server with HMR
npm run build  # produces dist/
npm run preview # serve dist/ locally to verify before deploy
```

## Comic Manifest (`comic.json`)

Each comic folder contains a `comic.json` that defines its metadata and panels.

```json
{
  "title": "Shadows of Lilim",
  "bgColor": "#1a1a2e",
  "panels": [
    { "src": "01.png" },
    { "src": "02.png", "bgColor": "#0f0f1a" },
    { "src": "03.png", "autoAdvanceMs": 400 },
    { "src": "04.png", "autoAdvanceMs": 200, "bgColor": "#000000" }
  ]
}
```

**Fields:**

| Field | Level | Required | Description |
| --- | --- | --- | --- |
| `title` | root | yes | Display name shown in gallery |
| `bgColor` | root | yes | Default background color for all panels |
| `panels` | root | yes | Ordered array of panel objects |
| `src` | panel | yes | Image filename relative to the comic folder |
| `bgColor` | panel | no | Overrides root bgColor for this panel only |
| `autoAdvanceMs` | panel | no | Auto-advances after N ms; no click needed |

The gallery uses `panels[0].src` as the cover image and `panels.length` for the panel count.

## Module API (`ClickComic`)

```js
import { ClickComic } from './click-comic.es.js'

// Mount and start
const comic = new ClickComic({
  container: document.getElementById('cutscene'), // required — DOM element to mount into
  src: '/comics/shadows-of-lilim/comic.json',     // required — URL to comic.json
  onComplete: () => {                              // optional — called once after last panel
    comic.destroy()
    resumeGame()
  }
})

comic.start()    // fetches manifest, preloads images, mounts viewer, begins playback
comic.destroy()  // unmounts viewer, removes all event listeners, cleans up state
```

**Behavior:**

- If `onComplete` is provided: comic stops after the last panel and calls `onComplete` once
- If `onComplete` is omitted: comic loops forever (standalone website mode)
- `destroy()` is safe to call at any time, including before `start()` resolves

## Viewer Page

**URL:** `viewer.html?comic=shadows-of-lilim`

- Full viewport, zero UI chrome — no progress bar, no title, no controls visible
- Page background color = active panel's `bgColor` (updates instantly on panel change)
- Panel image is centered, contained to a max-width, scaled to fit without cropping
- The `ClickComic` class is instantiated with the slug from the query string; `onComplete` is omitted so it loops

**Input handling (all advance to next panel):**

- Mouse click / tap anywhere
- Spacebar
- Right arrow / down arrow
- Swipe left

**Auto-advance:** Panels with `autoAdvanceMs` fire automatically; user input during auto-advance is ignored for that panel's duration.

## Gallery Page

**URL:** `index.html` (root)

- Minimal header: "Click Comics" in uppercase, spaced lettering
- Cover grid — responsive columns
- Each card:
  - Background = comic's root `bgColor`
  - Cover image = first panel (`01.png`)
  - Title and panel count below the image
  - Clicking navigates to `viewer.html?comic=<slug>`
- Gallery.js fetches `comics/index.json` listing all available comic slugs, then fetches each `comic.json` to render the cards
- `comics/index.json` is generated by a Vite plugin (`vite-plugin-comics-index`) that scans `public/comics/` at build start and during dev — no manual maintenance required

---

## Static Deployment

Deploy the `dist/` folder. No server-side logic required. Comics are static assets copied from `public/comics/`. Works on GitHub Pages, Netlify, any CDN.

## Embedding in a Game (Cutscene Mode)

```js
import { ClickComic } from './click-comic.es.js'

// Show cutscene, then resume game
function playCutscene(slug, onDone) {
  const container = document.getElementById('cutscene-layer')
  container.style.display = 'block'

  const comic = new ClickComic({
    container,
    src: `/comics/${slug}/comic.json`,
    onComplete: () => {
      comic.destroy()
      container.style.display = 'none'
      onDone()
    }
  })

  comic.start()
}
```

The module has no external dependencies and makes no assumptions about the surrounding application.

## Out of Scope (for now)

- Sound / music per comic or panel
- Animated transitions between panels (fade, wipe)
- Progress indicator
- Chapter / section grouping within a comic
- CMS or admin UI for managing comics
